Media Browser

User Interface > Controls

A control that presents content from a Media Server.

Properties

Property Description
Appearance The Appearance settings of the control.
ContainerCommandVariable A Variable used to execute commands on the current container. (See Commands)
ContainerIDVariable A Variable bound to the ID of the current container.
ContainerTitleVariable A Variable bound to the title of the current container.
GridLayout The Layout settings for the control when the View property is set to Grid or Auto.
Height The suggested height of the control.
Images A collection of images for the control. (See MediaObjectImage)
ItemCommandVariable A Variable that defines the command to execute when an music track or video is clicked. (See Commands)
Left The distance between the left side of the control and the left side of its parent Canvas.
ListLayout The Layout settings for the control when the View property is set to List or Auto.
Name The identifying name of the object.
Renderer The UPnP Media Renderer or Media Element that receives command requests such as PlayNow from the control.
Server The Media Server that provides content for the control.
Top The distance between the top of the control and the top of its parent Canvas.
View A value that determines how items are displayed in the control.
Visibility The Visibility settings of the control.
Width The suggested width of the control.
ZIndex The ZIndex determines the order on the z-plane in which the control appears.

Commands

Command Parameters Description
SetRenderer renderer Set the Media Renderer for the control at runtime. (See Remarks)
SetServer server, container Set the Media Server and container for the control at runtime. (See Remarks)
GoBack n/a Navigate to the parent container, if there is one.
PlayNow n/a Play an item or container immediately. If executed on a container, the container must be a valid playlist.
PlayNext n/a Play an item or container next. If executed on a container, the container must be a valid playlist.
SonosPlayNow n/a Add an item or container to the bottom of the queue and play it immediately.
SonosPlayNext n/a Add an item or container to the queue and play it next.
SonosAddToQueue n/a Add an item or container to the bottom of the queue.
SonosReplaceQueue n/a Remove all items from the queue and replace it with the current item or container.
SonosPlayTrack n/a Play an item that is currently in the queue.
SonosRemoveTrack n/a Remove an item that is currently in the queue.
SonosClearQueue n/a Remove all items from the queue.
SonosPlayLineIn n/a Play audio that is connected to the line-input.

Remarks

Commands are executed either by setting the ContainerCommandVariable or ItemCommandVariable to the desired command name. We recommend using Virtual Variables for these properties however that is not always necessary. Any commands that require parameters must include them as a standard URL query string: they must appear after a question mark '?', they must be separated by an ampersand '&', and the parameter/value pairs must be separated by an equal sign '='. As an example, if you want to change your server to 'Sonos Media Server' you would have to set the ContainerCommandVariable to 'SetServer?server=Sonos+Media+Server&container=0'. Remember, spaces are not valid in URL query strings. In its place either '+' or '%20' should be used. That is why the server parameter value in this example is 'Sonos+Media+Server' and not 'Sonos Media Server'. The root object ID for all Media Servers is '0'. If you don't know what container value to provide just set it to '0' as that will always be valid.

Most of these commands are demonstrated in our Sonos Example.
