Media Element

User Interface > Controls

Represents an object that renders audio and video to the display.

Properties

Property Description
AreTransportControlsEnabled A boolean value that indicates whether the media element shows system-provided playback controls.
AutoPlay A boolean value that indicates whether media will begin playback automatically when the control comes into view.
CommandVariable A Variable used to execute playback commands on the control. (See Commands)
Height The suggested height of the control.
Left The distance between the left side of the control and the left side of its parent Canvas.
Name The identifying name of the object.
Source The URL that provides video and/or audio to the media element.
SourceVariable A Variable bound to the Source property of the control.
Top The distance between the top of the control and the top of its parent Canvas.
TransportStateVariable A Variable bound to the current transport state of the control. (Closed, Opening, Playing, Paused, Stopped)
Visibility The Visibility settings of the control.
Width The suggested width of the control.
ZIndex The ZIndex determines the order on the z-plane in which the control appears.

Commands

Command Parameters Description
Pause n/a Pauses media at the current position.
Play n/a Plays media from the current position.
Stop n/a Stops and resets media to be played from the beginning.

Remarks

AreTransportControlsEnabled is only supported in the apps themselves. The simulator will not show playback controls when this property is set.

TransportStateVariable is read-only. To control playback please use the CommandVariable property.

Do not use this control to display a MJPEG stream from an IP camera. While it may work in the simulator, it will not in the apps themselves. Only use this control to display H.264 streams from IP cameras. You can define credentials for the connection in the Source URL like so:

rtsp://username:password@IPADDRESS:554/live.sdp
