HTTP Variable

Devices > Variables

Represents a variable that is used to execute a HTTP request.

Properties

Property Description
Content The contents of the HTTP message.
ContentType The content type of the HTTP message.
Method The HTTP method to be performed on the request URI.
Name The identifying name of the object.
StatusPath A JSONPath or XPath expression that points a specific value in the response string returned from a HTTP request. (See Remarks)
StatusSource The source for the variable's display value.
URI A URI relative to the source that defines the HTTP request.

Remarks

Variable values can be referenced in the Content and URI definitions.

To include the current value for a variable in the Content or URI, add the flag $HomeRemote.X.Value, where X is the name of your variable.

To include the requested value for a variable in the Content or URI, add the flag $HomeRemote.X.RequestedValue, where X is the name of your variable.

The requested value is the value that the variable is set to when a control changes. Consider the situation where variable X which is bound to the States property on either a Button or an ImageButton control. $HomeRemote.X.RequestedValue will be replaced with the OnValue (or OffValue) of the button when it is clicked.

The StatusPath property is only used when the StatusSource property is set to either StatusRequest or VariableRequest. When set to StatusRequest, the reponse data from the parent HttpClient's StatusRequest is the source for the variable's display value. When set to VariableRequest, the response from the request defined by this variable is the source for the variable's display value.
